# 输入输出示例

## 示例 1：GitHub 项目评估

### 输入

```
帮我评估这个项目：https://github.com/astral-sh/ruff
```

### 输出

```markdown
# 价值评估报告：《Ruff》

> **类型**：🐙 GitHub项目 | **评估时间**：2026-01-28

## 📊 总分：92/100

## 🎯 快速结论

| 项目 | 内容 |
|------|------|
| 一句话总结 | 用 Rust 编写的极速 Python Linter，性能远超传统工具 |
| 推荐指数 | ⭐⭐⭐⭐⭐（5/5）|
| 适合人群 | Python 开发者、追求代码质量的团队、CI/CD 优化需求者 |

## 📈 维度评分

| 维度 | 分数 | 说明 |
|------|------|------|
| 实用性 | 95/100 | 解决 Python 代码检查的核心痛点，兼容 Flake8/isort 规则 |
| 代码质量 | 90/100 | Rust 实现，架构清晰，测试覆盖完善 |
| 文档完善度 | 92/100 | 官方文档详尽，配置示例丰富 |
| 社区活跃度 | 95/100 | 40k+ Stars，Issue 响应快，持续更新 |
| 可维护性 | 88/100 | 依赖合理，但 Rust 门槛对贡献者有一定要求 |

## 📝 详细评价

### ✅ 优点
- 性能卓越：比传统工具快 10-100 倍
- 规则兼容：支持 Flake8、isort、pyupgrade 等多种规则
- 开发体验：单一二进制，零配置即可使用
- 社区活跃：Astral 团队全职维护

### ⚠️ 不足
- 部分冷门规则尚未覆盖
- Rust 代码库对 Python 用户贡献有门槛

### 💡 建议
- 强烈推荐替换现有的 Flake8 + isort 工具链
- 建议在 CI 中集成以加速代码检查流程
```

## 示例 2：书籍评估

### 输入

```
评估一下《Designing Data-Intensive Applications》这本书
```

### 输出

```markdown
# 价值评估报告：《Designing Data-Intensive Applications》

> **类型**：📚 书籍 | **评估时间**：2026-01-28

## 📊 总分：95/100

## 🎯 快速结论

| 项目 | 内容 |
|------|------|
| 一句话总结 | 数据系统领域的圣经级著作，深入浅出讲解分布式系统核心原理 |
| 推荐指数 | ⭐⭐⭐⭐⭐（5/5）|
| 适合人群 | 后端工程师、数据工程师、架构师、有 1-2 年经验的开发者 |

## 📈 维度评分

| 维度 | 分数 | 说明 |
|------|------|------|
| 实用性 | 92/100 | 涵盖实际系统设计中的核心问题和权衡 |
| 深度 | 98/100 | 深入原理层面，引用大量论文和实际案例 |
| 可读性 | 95/100 | 结构清晰，图表丰富，循序渐进 |
| 时效性 | 90/100 | 2017 年出版，核心原理不过时，部分工具有更新 |
| 作者权威性 | 100/100 | Martin Kleppmann，分布式系统领域知名专家 |

## 📝 详细评价

### ✅ 优点
- 体系完整：从存储到计算到一致性，形成完整知识框架
- 深入本质：不仅讲 What，更讲 Why 和 Trade-offs
- 案例丰富：大量真实系统案例（Kafka、Cassandra、Spanner 等）
- 引用扎实：每章末尾有详尽的参考文献

### ⚠️ 不足
- 篇幅较长（600+ 页），需要投入大量时间
- 部分章节对无分布式经验的读者有一定门槛

### 💡 建议
- 强烈推荐作为数据工程师/后端工程师的必读书目
- 建议结合实际项目经验阅读，效果更佳
- 可先读第一部分建立整体框架，再深入后续章节
```
